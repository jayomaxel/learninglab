# LinguistFlow AI 产品详细介绍文档 (v0.1.0)

## 1. 产品概述

**LinguistFlow AI**（又名 LearningLab）是一款专为高效语言学习设计的智能化 Web 应用。它结合了 **Google Gemini AI** 技术与现代教育心理学（如间隔重复、RSVP 阅读法），为语言学习者提供全方位的听、读、记体验。

该产品目前支持 **英语 (EN)**、**法语 (FR)** 和 **韩语 (KR)**，并深度集成 AI 能力，能够根据用户的 CEFR 等级（A0 到 C2）动态生成和分析学习内容。

---

## 2. 核心功能模块 (已实现)

### 2.1 任务中心 (Mission Center)
*   **每日目标追踪**：设定并追踪量化的单词记忆、听力及阅读目标。
*   **进度可视化**：直观展示当日成就，助力用户建立长效学习回路。

### 2.2 听力实验室 (Listening Lab)
*   **原生多模态素材生成**：利用 **Gemini 2.5 Flash TTS** 模型直接输出音频，实现高自然度的听力材料。
*   **自适应听写 (Flow Sync)**：系统实时感应用户录入节奏。当检测到输入间断超过 1.2s 时自动降速至 0.5x，超过 2.5s 时自动循环当前句段。
*   **逐词校验系统**：支持针对转录文本的实时拼写校验，正确显示绿色，错误显示红色。

### 2.3 极速阅读器 (Speed Reader)
*   **RSVP 阅读法**：采用快速串行视觉展示技术，通过 **ORP (最佳识别点)** 算法大幅提升阅读效率。
*   **智能素材抓取**：支持主题生成或解析 URL，针对不同语言等级进行语义分句。

### 2.4 智能生词本 (Vocabulary Bank)
*   **一键收藏与 AI 释义**：在听力或阅读中发现的生词，点击即可通过 AI 自动补全翻译及上下文例句。
*   **间隔重复复习 (SRS)**：基于改进的 SM-2 算法，自动计算单词复习周期。

## 3. 核心技术架构 (100% 真实落地)

### 3.1 自适应速率控制算法
*   **空闲检测机制 (Idle Detection)**：在 React 的 `requestAnimationFrame` 循环中记录 `lastKeystrokeTime`，实现非阻塞的语速动态切换（1.0x -> 0.5x -> Loop）。
*   **Loop Anchor (循环锚点)**：卡壳时自动回退至当前句首，通过无感重听降低学习门槛。

### 3.2 RSVP 与 ORP 对齐技术
*   **40% 偏移算法**：单词最佳识别点固定在长度的 40% 位置（长词偏左，短词居中）。
*   **Flexbox 居中对齐**：采用 `flex-1` 分栏布局，确保红色中心点在单词切换时物理位置绝对固定，消除视觉抖动。

### 3.3 离线优先与性能优化
*   **IndexedDB 存储引擎**：多表结构（用户、语料、日志、词库）实现全数据持久化。
*   **Web Worker 背景解析**：
    *   **PDF.js 集成**：通过独立 Worker 解析大体积 PDF 文件。
    *   **Cuckoo Filter (布谷鸟过滤器)**：在 Web Worker 中用 Cuckoo Filter 替代 Bloom Filter。支持动态删除单词（如用户从生词本中移出某词），且在相同假阳性率下比 Bloom Filter 具备更高的空间利用率。
*   **rAF 状态机优化**：在 `requestAnimationFrame` 循环中采用状态机模式。空闲检测仅在 `useRef` 中进行数值比较，只有当语速状态发生跨界跳变（如从 1.0x 切换至 0.5x）时才触发 React 渲染，彻底规避高频 `setState` 导致的性能瓶颈。

### 3.4 音频工程与 Flow Sync 优化
*   **变速不变调 (Pitch-Preserving)**：显式开启 `audioElement.preservesPitch = true`。在 Flow Sync 自动降速（1.0x -> 0.5x）时，确保人声频率不随采样率变化而降低，保持原生的语感与清晰度，这对于语言学习中的语音模仿至关重要。
*   **智能化音频缓存 (Audio Caching)**：引入 **Cache API** 机制。系统自动根据生成文本生成唯一 Key，并将 AI 输出的音频 Blob 持久化存储。用户在重听句子或循环锚点时，直接从本地缓存读取 ArrayBuffer，零延迟响应且节省 API 配额。

### 3.5 RSVP 算法演进 (已实现)
*   **标点中立 ORP (Punctuation-Neutral ORP)**：系统在计算最佳识别点 (ORP) 前，自动剥离标点符号算出核心词长度定中心位，渲染时再恢复。这解决了因标点导致红色中心点跳动的视觉疲劳问题。
*   **原生 DOM 驱动渲染 (High-Perf RSVP)**：在 WPM > 400 的高速场景下，避开了 React 虚拟 DOM 的 Diff 性能瓶颈。通过 `useRef` 直接操作原生 DOM 节点和 `requestAnimationFrame` 驱动更新，确保在高频刷新下无丢帧闪烁。
*   **动态停顿 (Dynamic Pacing)**：引入基于标点与词长的延迟乘数。遇到逗号时长延长 2 倍，句号/问号延长 3 倍，字符数 > 8 的长词延长 1.2 倍。这模拟了人类大脑在句法边界 (Syntactic Boundaries) 的认知缓冲。
*   **韩语 (KR) 形态素对齐**：针对韩语黏着语特性，算法将 ORP 焦点从传统的 40% 前移，确保视觉落点始终在语义核心的“词根”而非助词上。

## 4. 技术路线图 (Technical Roadmap - 开发中)

为了进一步提升产品力，以下特性已进入开发计划：

### 4.1 音频工程增强
*   **音轨权重校准**：计划针对法语连读 (Liaison) 和韩语字核 (Syllable) 引入更精细的时长权重系数。
*   **智能滤波 (Noise Gate)**：计划引入噪音门限节点，过滤 LLM 偶尔生成的背景幻觉杂音。

### 4.2 性能与系统级集成
*   **iOS 性能优化**：集成 MediaSession API，解决移动端 Safari 锁屏播放与性能降级策略。
*   **非阻塞分级控制 (已实现)**：基于 **Gemini Structured Outputs**，在素材生成阶段同步输出 `hardWords` 及其索引位。UI 层直接渲染下划线标记，彻底替代了昂贵的客户端文本匹配与二次 AI 释义请求。

### 4.3 分布式一致性
*   **逻辑时钟 (HLC) 同步**：针对多设备离线复习的场景，引入混合逻辑时钟处理数据合并冲突。

### 4.4 安全与 API 管理 (Security Roadmap)
*   **后端请求代理 (AI Proxy)**：针对 API Key 泄露风险，实施由前端发起的匿名化请求，由后端 Node.js 服务透传至 Gemini API。后端负责 API 鉴权、速率限制 (Rate Limiting) 及密钥动态轮转，避免前端抓包。
*   **动态密钥注入 (BYOK)**：支持用户自定义 API Key 并加密存储于 `localStorage`，作为开发实验室模式的替代方案。

### 4.5 Flow Sync 算法极致演进 (已实现)
*   **动态基线 (Dynamic Baseline)**：系统自动在用户输入前 10 个词时计算实时击键间隔 (WPM)。将阈值从死板的固定值升级为 `Avg_Interval * 1.5` 触发 0.5x，`Avg_Interval * 3` 触发 Loop。这消除了 PC 端与手机虚拟键盘输入速度差异带来的误判率。
*   **智能静音锚点 (Silence-Aware Loop Anchor)**：引入 Web Audio API 预分析音频振幅。系统将循环回退点 (Loop Anchor) 自动对齐至最近的静音帧 (Silence Threshold)，完美解决了循环重听时因切割时间戳导致的“断头词”(/s/ 等辅音丢失) 问题。

### 4.6 词库秒级去重 & 动态布谷鸟过滤器 (已实现)
*   **解决死锁瓶颈**：针对 Cuckoo Filter 装载率过高（>95%）时产生的踢出 (Kick-out) 死循环问题，实现了自动扩容机制。当当前滤表装载率达到 **85%** 时，后台自动新建容量翻倍的新滤表。
*   **分层查询性能**：支持百万级词库的秒级去重与动态删除，空间利用率与查询性能维持在最优状态，消除了用户长时间学习后系统变慢的隐患。

### 4.7 RSVP 2.0 极致性能 (已实现)
*   **原生 DOM 驱动渲染**：在 WPM > 400 的高速场景下，避开了 React 虚拟 DOM 的 Diff 性能瓶颈。通过 `useRef` 直接操作原生 DOM 节点和 `requestAnimationFrame` 驱动更新，确保在高频刷新下无丢帧闪烁。
*   **标点中立 ORP**：在计算最佳识别点 (ORP) 前自动剥离标点符号算出中心位，解决了因标点导致红色中心点跳动的视觉疲劳问题。

## 5. 总结

LinguistFlow AI 目前已完成了从 **AI 模态接入** 到 **核心交互算法** 的闭环。它不仅仅是一个实验室 Demo，而是一个架构稳健、在音频控制与离线数据处理方面具备独特工程实现的学习平台。
